WebORB Enterprise - Standalone Mode

Introduction

WebORB Enterprise Edition supports the standalone execution mode thus eliminating the requirement for a web server. Flex/Flash applications can connect to WebORB using messaging API. Flex applications can perform remoting invocations of the .NET assemblies deployed into WebORB via messaging channel.

Running WebORB Enterprise

To run the product standalone open a command prompt window, change current directory to /Inetpub/wwwroot/weborb30 and run weborbee.exe:

Notice the default port the product listens for requests is 2037. The port can be changed using the -port argument for the weborbee.exe product launcher. Once the product is started Flex and Flash clients can connect to it using the standard ActionScript API.

Flex Remoting over Messaging

To configure Flex clients to perform remoting invocations via messaging channel, change the channel name for the configured destinations to "weborb-rtmp" in remoting-config.xml:

<default-channels>
    <channel ref="weborb-rtmp" />
</default-channels>

If the change is made after Flex Builder project is created, make sure to reload your Flex project since Flex Builder caches and does not refresh server configuration. No additional configuration or special programming consideration needed, as Flex runtime automatically re-routes all remoting requests via the "weborb-rtmp" channel.
