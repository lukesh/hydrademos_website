Oops, something went wrong...
 		  

 

WebORB Management Console attempted to connect to WebORB in your virtual directory and received an error. There may be several reasons for the error. Most common are:
  • ASP.NET is not installed in IIS or not registered with the virtual directory - Make sure you can run ASP.NET applications. Try to open the following URLs:

    diagnostics.aspx
    weborb.aspx?diag
     
  • IIS metabase mappings for ASP.NET must be reconfigured - When you run the URLs above you may see an error referencing IIS metabase mappings. To solve the problem, open a command prompt window and change the directory to the .NET framework installation directory (for example \WINDOWS\Microsoft.NET\Framework\v2.0.50727). Run the following command:

    aspnet_regiis -i
     
  • Virtual directory is configured to use .NET 1.1 - Open virtual directory properties and make sure the ASP.NET version for the directory is 2.0 or above.
